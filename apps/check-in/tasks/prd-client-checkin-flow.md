# PRD: Client Self Check-In App

## Introduction

The Client Self Check-In App is a **digital receptionist** that enables salon and spa clients to check themselves in upon arrival. It's designed to be as friendly, intuitive, and intelligent as the best human receptionistâ€”greeting clients warmly, remembering their preferences, and guiding them effortlessly through the check-in process.

**The Ultimate Goal:** Replace the front desk receptionist entirely for the check-in process, freeing staff to focus on client service rather than administrative tasks.

Clients can use a store tablet/kiosk or their personal phone to identify themselves, confirm or modify appointments, select services (for walk-ins), choose their preferred technician, and add guestsâ€”all in under 60 seconds.

## Goals

- **Replace receptionist:** Handle 100% of check-in interactions without staff intervention
- **Speed:** Complete check-in flow in under 60 seconds for existing clients
- **Simplicity:** Intuitive enough for all ages without assistance
- **Intelligence:** Detect walk-in vs. appointment and adapt flow accordingly
- **Flexibility:** Support tablet kiosk, iPad, Android tablet, and mobile web
- **Engagement:** Display promotions and loyalty points to drive business
- **Accuracy:** Ensure correct service and technician matching

## Design Philosophy: The Intelligent Receptionist

Think of this app as a **friendly, intelligent receptionist** who:

1. **Greets warmly** - Knows the time of day, recognizes returning clients by name
2. **Remembers everything** - Pulls up appointments, knows preferences, tracks loyalty
3. **Guides effortlessly** - Makes the next step obvious, never confuses
4. **Communicates clearly** - Shares promotions, wait times, important info
5. **Handles exceptions** - Adapts to walk-ins, appointment changes, groups
6. **Never rushes** - Patient, calm, always helpful

The UI must be **so intuitive that a first-time visitor knows exactly what to do** within 2 seconds of looking at the screen.

### US-001: Welcome Screen - The Digital Receptionist Face
**Description:** As a client arriving at the salon, I want to see a welcoming, branded screen that clearly shows me how to check in and informs me of any current promotions.

**Design Note:** ğŸ¨ **This screen must be designed using the Mango Designer skill** to ensure world-class UI/UX that makes the check-in action immediately obvious.

**Acceptance Criteria:**
- [ ] **Hero section** with store logo, name, and welcoming message
- [ ] **Time-based greeting:** "Good Morning!", "Good Afternoon!", "Good Evening!"
- [ ] **Promo banner area:** Configurable space for salon to display:
  - Current promotions (e.g., "20% off first visit!")
  - Announcements (e.g., "Now offering lash extensions!")
  - Seasonal messages (e.g., "Happy Holidays from our team!")
- [ ] **Two prominent CTAs** that dominate the screen:
  - "Check In with Phone Number" (primary)
  - "Scan QR Code" (secondary)
- [ ] **Visual hierarchy:** A client should know what to do within 2 seconds
- [ ] **First-time visitor link:** Subtle "New here? Sign up" option
- [ ] **Idle behavior:** Return to this screen after 60 seconds of inactivity
- [ ] **Kiosk-friendly:** No navigation chrome, full-screen experience
- [ ] `npm run typecheck` passes
- [ ] Verify in browser using dev-browser skill

**Design Requirements for Mango Designer:**
- Clean, modern, spa-like aesthetic
- Calming color palette with brand accent colors
- Large, impossible-to-miss CTA buttons
- Promo area that's visible but doesn't compete with CTAs
- Works in both portrait and landscape tablet orientations
- Feels like walking up to a friendly receptionist

## Functional Requirements

### Check-In Flow
- **FR-1:** System must support two identification methods: phone number entry and QR code scan
- **FR-2:** Phone number keypad must auto-format input as (XXX) XXX-XXXX
- **FR-3:** System must query client database and determine: new client, existing walk-in, or existing with appointment
- **FR-4:** System must detect appointments scheduled for today and route to appointment confirmation flow
- **FR-5:** New client registration must require only: first name, last name, phone number
- **FR-6:** Email and zip code fields must be optional for new client signup

### Appointment Handling
- **FR-7:** System must display booked appointment details for clients with today's appointments
- **FR-8:** System must allow appointment confirmation without changes
- **FR-9:** System must support appointment editing based on salon-configurable policy
- **FR-10:** Editable fields may include: services, technician, time (per salon policy)
- **FR-11:** All edits must validate against real-time availability before confirming
- **FR-12:** System must handle multiple appointments for same client on same day

### Service Selection (Walk-In)
- **FR-13:** Services must be displayed as visual cards with name, duration, and price
- **FR-14:** Services must be filterable by category
- **FR-15:** System must support multi-select for services
- **FR-16:** Selected services summary must show count, total duration, and total price

### Technician Selection
- **FR-17:** Only ONE technician may be selected per client check-in
- **FR-18:** Technicians must be filtered by specialty matching selected services
- **FR-19:** "Next Available" must be the default selection
- **FR-20:** Technician cards must display real-time availability status
- **FR-21:** Availability status must show: Available (green), Wait time estimate (yellow), Busy (red)

### Guest Management
- **FR-22:** System must support adding unlimited guests to a single check-in
- **FR-23:** Guest registration must require only: first name, last name
- **FR-24:** Guest phone number must be optional with clear "Skip" option
- **FR-25:** Each guest must complete service and technician selection (1 tech per guest)
- **FR-26:** Guests earn loyalty points and are tracked in the system

### Confirmation & Queue
- **FR-27:** System must generate unique check-in number (format: A-XXX)
- **FR-28:** System must calculate and display queue position
- **FR-29:** System must display client's loyalty points balance
- **FR-30:** System must show visual progress toward next reward
- **FR-31:** Staff must receive real-time queue updates in POS

### Notifications
- **FR-32:** System must send SMS notification when technician is ready
- **FR-33:** SMS must include client name, technician name, and store name
- **FR-34:** SMS sent to guests with phone numbers on file

### Promotions
- **FR-35:** Welcome screen must display configurable promotional content
- **FR-36:** Promotions must be manageable from Control Center
- **FR-37:** Promotions must support scheduling (start/end dates)

### Error Handling
- **FR-38:** All screens must have back navigation (except welcome and final)
- **FR-39:** All error states must provide "Try Again" and "Start Over" options
- **FR-40:** Network errors must be detected and displayed within 5 seconds
- **FR-41:** Session must timeout after 60 seconds of inactivity
- **FR-42:** Offline check-ins must be queued and synced when network restored

## Non-Goals (Out of Scope for V1)

- **No future appointment booking:** This is check-in only, not scheduling
- **No payment processing:** Payment handled at checkout
- **No loyalty point redemption:** Points are display-only
- **No client history view:** Past appointments not shown during check-in
- **No real-time wait time updates:** Estimate is static per check-in
- **No multi-location support:** Single store check-in only
- **No multi-language:** English only for V1
- **No photo upload:** Client photos managed elsewhere
- **No waitlist for busy times:** Future enhancement
