## 1. Overview

The Reports Module provides real-time business intelligence through daily sales summaries, staff performance metrics, and transaction analytics. Designed for quick scanning during busy periods and detailed analysis during slow times.

### Design Goals

| Goal | Description |
|------|-------------|
| **Scannable** | Key metrics visible in < 2 seconds |
| **Accurate** | Real-time data, clear timestamps |
| **Actionable** | Insights lead to decisions |
| **Exportable** | Easy to share with accountants |
| **Offline-Ready** | Cached view when offline |

## 2. Design Philosophy

### Core Principles

1. **Numbers First** - Large, prominent metrics
2. **Visual Hierarchy** - Most important data largest
3. **Trend Indicators** - Up/down arrows show direction
4. **Comparative Context** - vs. yesterday, vs. last week
5. **Offline Transparency** - Clear when data is stale

### Visual Language

- **Metric Cards** - Big numbers with context
- **Clean Charts** - Minimal axes, clear data
- **Status Colors** - Green up, red down
- **Muted Details** - Supporting info smaller

## 8. Accessibility

### 8.1 WCAG 2.1 AA Requirements

| Requirement | Specification |
|-------------|---------------|
| Color Contrast | 4.5:1 for text |
| Chart Alt Text | Screen reader descriptions |
| Focus Indicators | 2px solid outline |
| Data Tables | Proper headers and labels |

### 8.2 Screen Reader Labels

| Element | Label |
|---------|-------|
| Metric card | "{label}: {value}, {comparison}" |
| Chart | "Hourly revenue chart, peak at {time} with {amount}" |
| Bar | "{time}: {amount}" |
| Ticket row | "Ticket {number}, {client}, {amount}, {method}" |

<svg role="img" aria-label="Hourly revenue chart showing peak at 1 PM with $1,180">
